Spring Boot Compatibility

MockServer targets Java 11 as the minimum supported version and uses the javax namespace throughout. This means MockServer is compatible with Spring Boot 2.x (which uses Spring Framework 5.x) but is not compatible with Spring Boot 3.x (which uses Spring Framework 6.x).

Spring Boot 3 requires Java 17+ and migrated from the javax namespace to the jakarta namespace. If your project uses Spring Boot 3, you can still use MockServer as a standalone Docker container or process, but the Spring Test Execution Listener and JUnit integrations that depend on Spring classpath integration are not supported.

Spring Boot Version Spring Framework Namespace MockServer Compatible
2.x 5.x javax Yes
3.x 6.x jakarta No (use Docker or standalone)
 

Upgrading MockServer

When upgrading MockServer between versions, be aware of the following changes that may affect your setup:

  • Java 11 minimum (since 5.14.0) - MockServer requires Java 11 or later. If you are running on Java 8, you must upgrade your JVM before upgrading MockServer.
  • JSON Schema validator version change - the underlying JSON Schema validation library was updated in recent versions. If you use JSON Schema body matchers with custom schemas, verify they still validate correctly after upgrading.
  • Expectation JSON format - the JSON format for expectations has evolved between major versions. If you use JSON expectation initializer files (MOCKSERVER_INITIALIZATION_JSON_PATH), test your initializer files against the new version before deploying. Fields that were previously accepted may be rejected or interpreted differently.

It is recommended to run your full test suite after upgrading to catch any compatibility issues early.

 

HTTP/2 Proxy Limitations

MockServer can accept HTTP/2 connections and serve mock responses over HTTP/2. However, when MockServer is used as a proxy and forwards requests to downstream services, HTTP/2 requests are downgraded to HTTP/1.1. This means:

  • Mocking with HTTP/2 - works correctly. Clients can connect to MockServer using HTTP/2 and receive mock responses.
  • Proxying with HTTP/2 - the forwarded request to the downstream service will be sent as HTTP/1.1, regardless of the original protocol version. The response returned to the client will also be HTTP/1.1.

If your tests require end-to-end HTTP/2 proxying (including HTTP/2 to the upstream server), this is not currently supported. Consider using MockServer in mock mode instead of proxy mode for HTTP/2 endpoints.

 

Single Port Architecture

MockServer serves both mock and proxy traffic on the same port (default 1080). There is no separate port for proxying. The proxyRemotePort and proxyRemoteHost settings specify the upstream target server that unmatched requests are forwarded to — they do not open a separate listening port. All expectations, proxy forwarding, and protocol support (HTTP, HTTPS, SOCKS) share the single serverPort.
